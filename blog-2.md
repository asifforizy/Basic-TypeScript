How do Pick and Omit utility types prevent code duplication while creating specialized "slices" of a master interface? Discuss how this keeps your code DRY (Don't Repeat Yourself).

answer:
Pick and Omit create new types from an existing type instead of rewriting it. Pick select only needed things and omit removes unnecessary things. they prevent code duplication because you dont need to write the same code again and again. this keeps you code dont repeat yourself (DRY) .
